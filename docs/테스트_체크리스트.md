# 실기기 테스트 체크리스트

**목적:** 앱스토어 심사 제출 전 Live Activities 및 Core Motion 기능 검증

---

## 테스트 환경 준비

### 필요 기기:
- [ ] **iPhone 14 Pro/Pro Max/Plus** (Dynamic Island 포함) — 필수
- [ ] **iPhone 13** (Dynamic Island 미포함) — 권장
- [ ] **iPhone SE** (구형, 잠금 화면 업데이트 배너만 표시) — 선택사항

### 준비물:
```bash
# 1. 최신 빌드 생성
flutter clean
flutter pub get
flutter build ios --release

# 2. Xcode에서 실기기로 빌드
# Product → Destination → 실제 기기 선택
# Product → Run (⌘R)

# 3. 콘솔 로그 확인
# Xcode: View → Debug Area → Show Debug Area (⌘⇧Y)
# 또는 Console 탭에서 "[MotionFitnessDebug]", "[LiveActivityDebug]" 필터링
```

## 테스트 시나리오

### 시나리오 1: 동작 및 피트니스 권한 허용 + Live Activity 시작

**사전 조건:**
- 앱 초기 설치 (또는 App Group 데이터 삭제)
- iPhone 14+ (Dynamic Island)

**실행 단계:**

| # | 행동 | 예상 결과 | ✓ |
|----|------|---------|---|
| 1 | 앱 실행 | 홈 화면 로드 | [ ] |
| 2 | "시작" 버튼 클릭 | 성경 말씀 선택 바텀 시트 표시 | [ ] |
| 3 | 말씀 1개 선택 + 테마 선택 | "동의" 버튼 활성화 | [ ] |
| 4 | "말씀과 함께 일과 시작하기" 클릭 | Soft Prompt 팝업 표시 (바텀 시트 설명) | [ ] |
| 5 | 팝업에서 "동의" 클릭 | iOS 기본 동작 및 피트니스 권한 팝업 표시 | [ ] |
| 6 | 권한 팝업에서 "확인" 선택 | 권한 팝업 닫힘, 앱으로 복귀 | [ ] |
| 7 | 200ms 대기 | **잠금 화면에 파란색 바 표시** ← 핵심 | [ ] |
| 8 | 잠금 화면 확인 | 구절 + 참조 + **"👣 0"** 표시 | [ ] |
| 9 | 30초 대기 (앱 배경 실행) | Core Motion 데이터 동기화 | [ ] |
| 10 | 잠금 화면 다시 확인 | **"👣 1234"** (실제 오늘 걸음 수) 업데이트됨 | [ ] |

**검증 포인트:**
- ✅ 초기 "👣 0"에서 실제 걸음 수로 변경됨 (Core Motion 동작 증거)
- ✅ Xcode 콘솔에 `[MotionFitnessDebug] fetchTodaySteps success:` 로그 없음 (DEBUG 조건화됨)
- ✅ 앱이 배경에서도 걸음 수 업데이트 계속됨

### 시나리오 2: Dynamic Island 상태 확인

**사전 조건:**
- iPhone 14+ (Dynamic Island)
- 시나리오 1 완료 (Live Activity 활성)

| # | 행동 | 예상 결과 | ✓ |
|----|------|---------|---|
| 1 | 잠금 화면에서 상태 바 관찰 | **상태 바에 동적 아일랜드 표시 안 됨** (최소 모드는 화면 상단 고정) | [ ] |
| 2 | 상태 바 (Dynamic Island) 길게 터치 | **Expanded View:** 전체 구절 + 걸음 수 표시 | [ ] |
| 3 | 확장 뷰에서 텍스트 읽음 | 구절이 2-3줄로 명확하게 표시됨 | [ ] |
| 4 | 확장 뷰 닫기 (바깥쪽 터치) | **Compact View:** 좌측(📖) + 우측(👣 1234) | [ ] |
| 5 | 30초 더 대기 | **우측 숫자 증가** (걸음 수 실시간 업데이트) | [ ] |

**검증 포인트:**
- ✅ Compact View에서 걸음 수가 실시간 변함 (지속적 갱신)
- ✅ Expanded View에서 구절이 잘려 보이지 않음

### 시나리오 3: 동작 및 피트니스 권한 거절 + 타이머 Fallback

**사전 조건:**
- 앱 완전 삭제 (`Settings → General → iPhone Storage → Scripture → 삭제`)
- App Group 데이터 삭제 (개발자 도구 또는 iCloud 드라이브)

**실행 단계:**

| # | 행동 | 예상 결과 | ✓ |
|----|------|---------|---|
| 1 | 앱 재설치 | 홈 화면 | [ ] |
| 2 | "시작" 버튼 → 말씀 선택 → "동의" | 이번엔 권한 팝업 표시 | [ ] |
| 3 | 권한 팝업에서 **"허용 안 함"** 선택 | 팝업 닫힘, 앱 복귀 | [ ] |
| 4 | 200ms 후 잠금 화면 확인 | **"⏱️ 00:00:30"** (타이머) 표시 | [ ] |
| 5 | 30초 더 대기 | **"⏱️ 00:01:00"** (타이머 증가) | [ ] |

**검증 포인트:**
- ✅ 권한 거절해도 **에러 없이** Live Activity 작동
- ✅ 걸음 수("👣")가 아닌 **타이머("⏱️")**로 전환됨
- ✅ 초 단위로 지속적으로 업데이트됨 (동적 데이터)

## 문제 해결

### 문제: Live Activity가 나타나지 않음
```
해결:
1. App Group 확인: com.jgh.malsseumdonghaeng 활성화?
   → Xcode: Signing & Capabilities → App Groups
2. NSSupportsLiveActivities = true 확인 (Info.plist)
3. ActivityKit Framework linked 확인
4. Xcode 콘솔: "[LiveActivityDebug] Failed to start" 로그 확인
```

### 문제: 걸음 수가 업데이트되지 않음
```
해결:
1. 동작 및 피트니스 권한 허용 확인
   → Settings → Privacy & Security → Motion & Fitness
2. MotionFitnessService.shared.startObserving() 호출 확인
3. pedometer.startUpdates() 성공 로그 확인
4. 1분 이상 대기 (배경 데이터 동기화)
```

### 문제: 타이머가 표시되지 않음 (권한 거절 시)
```
해결:
1. AppDelegate.swift의 isAuthorized 값 확인
2. contentState.useTimer = true 설정 확인
3. ScriptureDynamicDataView에서 "⏱️" 렌더링 확인
```
