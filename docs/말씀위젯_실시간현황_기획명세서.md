# 말씀 위젯 앱 (Live Activities) 기획 및 개발 명세서

## 1. 기획 배경 및 히스토리 (History & Context)

### 🎯 초기 목표
* **사용자 니즈:** 기존의 작고 답답한 일반 위젯(WidgetKit)을 벗어나, 내비게이션 앱처럼 잠금화면 하단에 파란색 바(Bar) 형태로 크고 압도적으로 성경 말씀을 띄우고자 함.
* **기술 스택 확정:** 이를 구현할 수 있는 유일한 UI인 iOS의 `ActivityKit` (Live Activities - 실시간 현황) 채택.

### ❌ 논의된 대안과 거절 사유 (Rejected Ideas)
앱의 방향성을 잡아가는 과정에서 아래와 같은 기획들이 논의되었으나, 명확한 한계로 인해 폐기됨.

1. **강제적인 '묵상 루틴' 도입 (수명 연장용)**
   * *내용:* Live Activities의 8시간 시간 제한을 피하기 위해, 사용자가 하루 3번 직접 버튼을 눌러 위젯을 켜게 만드는 기획.
   * *거절 사유:* 시간 제한이 없는 안드로이드 사용자에게까지 OS 제약으로 인한 불편한 UX를 강제하게 됨. 양대 OS 간의 앱 일관성을 심각하게 저해하므로 기각.

2. **기본 잠금화면 위젯(WidgetKit)으로 선회**
   * *내용:* 24시간 내내 서버 없이 영구적으로 띄우기 위해, 유버전(YouVersion) 앱처럼 일반 위젯을 사용하는 방안.
   * *거절 사유:* 잠금화면 시계 아래에 들어가는 위젯은 크기가 너무 작아 초기 목표였던 '압도적인 존재감'과 가독성을 만족시킬 수 없어 기각.

3. **서버(푸시 알림)를 통한 24시간 무한 연장 꼼수**
   * *내용:* Supabase 등 무료 백엔드를 구축해 주기적으로 백그라운드 푸시를 쏴서 8시간 제한을 무한히 연장하는 방안.
   * *거절 사유:* 애플 앱스토어 심사(App Review)에서 **'API 오용(Misuse of API)'으로 99% 리젝(거절)**됨. 애플은 Live Activities를 정적인 텍스트 노출용으로 영구 사용하는 것을 엄격히 금지함.

### ✅ 최종 기획 도출의 전환점
* 단순 텍스트 띄우기가 아닌 **"일과 시작 시점에 켜서 8시간 동안 말씀과 동행하는 세션"**으로 개념을 완전히 재정의함.
* 심사 통과를 위한 명분과 사용자 가치를 동시에 잡기 위해 **'만보기(실시간 걸음 수)' 데이터**를 결합하기로 합의.

## 2. 최종 결정 사항
* **기능 컨셉:** '일과 시간 동안 말씀과 함께 걷기' (최대 8시간 라이브 세션)
* **적용 기술:** iOS `ActivityKit` (Live Activities)
* **운영 환경:** **서버 없음 (Local-Only)**. 운영 비용 0원 및 완전한 오프라인 동작 보장.
* **동적 데이터 (심사 우회 및 가치 창출):** `Core Motion (CMPedometer)` 연동을 통한 **만보기(실시간 걸음 수) 표시**. 이를 통해 단순 정적 텍스트 배치가 아닌 '실시간 변화하는 사용자 활동 추적 이벤트'로 규정하여 심사 가이드라인 충족.

## 3. UI/UX 레이아웃 가이드
### 메인 잠금화면 바 (Lock Screen Bar)
* **배경:** 유저가 선택한 테마 색상 (앱 내에서 파란색, 베이지색, 다크모드 등 커스텀 가능).
* **좌측 상단:** "오늘의 일과 진행 중" 또는 "말씀과 함께 걷기" (세션의 목적을 명시하는 컨텍스트).
* **중앙~좌측:** 오늘의 성경 구절 (가독성을 위해 2~3줄 이내로 크고 명확하게 배치).
* **우측 하단:** `👣 3,420 걸음` (화면에서 지속적으로 변하는 '동적 데이터' 앵커포인트). *단, 권한 거절 시 대체 데이터 표기 (8항 참조).*

## 4. 다이내믹 아일랜드 (Dynamic Island) 상세 기획
*iOS 가이드라인에 따라 Live Activities 구현 시 다이내믹 아일랜드 3가지 상태 대응은 **필수(Mandatory)**.*

* **Compact View (평상시 좌우 분리):**
  * **디자인:** 좌측에는 말씀 아이콘(📖), 우측에는 현재 걸음 수(👣 3.4k) 배치.
  * **애니메이션 제약 (Marquee 불가):** 미디어 재생 앱(음악, 팟캐스트)에서 보이는 텍스트 흐름 효과(Marquee)는 애플의 `NowPlaying` 시스템 API 전용 특권임. 커스텀 `ActivityKit` 앱에서 이를 억지로 구현하려 하면 시스템 리소스 낭비로 리젝되거나 데이터 갱신 제한(Budget Limit)에 걸리므로, **정적인 고정 텍스트 배치로 타협**함.
* **Expanded View (길게 터치 시 확장):** * **디자인:** 잠금화면 위젯과 유사한 비율로 확장되며 구절 전체와 걸음 수 동시 노출. 터치 기반 액션이므로 구절 전체를 안전하게 렌더링 가능.
* **Minimal View (다른 앱과 충돌 시 작은 원):** * **디자인:** 성경 아이콘(📖) 단독 노출.
  * **개인화 기능:** 사용자가 앱 내 설정에서 아일랜드 최소화 상태에 띄울 아이콘(예: 📖, ✝️, 👣, ❤️ 등)을 직접 선택 가능하도록 `ContentState`로 분기 처리.

## 5. 앱스토어 심사(App Review) 핵심 주의사항
* **정확한 `Info.plist` 권한 문구:** 동작 및 피트니스 접근 권한 요청 시, *"일과 시간 동안 말씀과 동행하며 걸은 걸음 수를 잠금화면에 표시하기 위해 동작 및 피트니스 데이터에 접근합니다."* 처럼 사용 목적을 구체적으로 서술해야 함.
* **심사역 메모 (Review Notes) 작성 전략:** 앱 제출 시 리뷰어에게 이 기능의 성격을 명확히 어필.
  * *(예시)* "This Live Activity is an 8-hour daily session tracker. It tracks the user's real-time step count via Core Motion (CMPedometer) during their working hours. If permission is denied, it seamlessly falls back to displaying the elapsed 'Session Time' using a live timer, ensuring the Activity remains dynamic and functional."

## 6. 기술적 참고 사항 (개발 한계점 인지)
* **8시간 생명 주기 (Hard Limit):** 실시간 현황은 활성화 후 8시간이 지나면 OS가 강제 종료함. (만보기를 넣어도 이 제한 시간은 늘어나지 않음). 이를 '일과 종료'라는 자연스러운 흐름으로 사용자에게 인지시키는 UX/UI 기획이 필수적임.
* **걸음 수 업데이트 빈도:** 걸음 수를 1보 단위로 즉각 갱신하려고 무리하게 앱을 깨우면 배터리 소모 및 시스템 제약에 걸림. 시스템이 허용하는 적절한 주기(예: 화면이 켜질 때나 시스템 배치를 통한 간헐적 갱신)로 부드럽게 업데이트하는 방식을 사용해야 함.

## 7. 크로스 플랫폼 및 구형 기기 대응 방안 (Fallback Strategy)

### 📱 다이내믹 아일랜드 미지원 iOS 기기 (iPhone 13 이하, SE 등)
* **작동 원리:** OS 차원에서 자동 Fallback 처리됨. 별도의 분기 처리 코드 불필요.
* **잠금화면:** 최신 기기와 100% 동일한 Live Activity 레이아웃 노출.
* **폰 사용 중:** 상단 고정 UI 대신, 데이터 갱신 시 화면 상단에 일시적인 드롭다운 배너(Banner) 형태로 나타남.

### 🤖 안드로이드(Android) 플랫폼 대응
* **핵심 기술:** `Foreground Service`와 `Persistent Notification` (고정 알림) 결합.
* **잠금화면 / 알림창 (Live Activity 대체):** `RemoteViews`를 활용해 iOS의 파란색 바 디자인(말씀 + 만보기)을 알림창에 커스텀 레이아웃으로 동일하게 구현.
* **다이내믹 아일랜드 대체:** * 상단 상태 표시줄(Status Bar)에 앱 심볼(📖 또는 👣)을 고정 노출하여 백그라운드 활성 상태 인지.
  * *주의:* '다른 앱 위에 그리기(플로팅 뷰)' 권한을 사용한 대체는 유저 피로도 및 권한 획득의 어려움이 있으므로 도입하지 않음.

## 8. 앱 타겟 버전 및 호환성 정책
* **최소 지원 버전 (iOS Deployment Target):** **iOS 16.2 이상**
* **결정 사유:** Live Activities 및 Dynamic Island의 완벽한 제어(특히 백그라운드 데이터의 빈번한 갱신)를 위한 필수 API가 iOS 16.2부터 제공됨. 하위 버전(iOS 14~16.1) 호환을 포기함으로써 분기 처리로 인한 코드 복잡도를 줄이고 클라이언트 앱의 안정성을 극대화함.

## 9. 권한 요청 UX 및 우회(Fallback) 처리
사용자가 동작 및 피트니스 데이터 권한 요청에 당황하거나 거절하여 발생할 수 있는 앱심사 리젝 및 UX 훼손을 방지하기 위한 전략.

* **사전 안내 (Soft Prompt) 필수 적용:**
  * 기능 실행 버튼 클릭 시, OS 기본 권한 팝업을 즉시 띄우지 않고 앱 내부 커스텀 바텀 시트(Bottom Sheet)를 먼저 노출함.
  * *메시지 예시:* "잠금화면에서 8시간 동안 말씀과 동행하며 걸음 수를 기록합니다. 이를 위해 동작 및 피트니스 데이터 연동이 필요합니다."
* **버튼 컨텍스트 변경:** 단순한 '위젯 고정하기'가 아닌 **'말씀과 함께 일과 시작하기'**로 변경하여 세션 시작의 명분을 부여함.
* **권한 거절 시 이중 안전장치 (Fallback Timer):**
  * 사용자가 권한을 거절하더라도 에러 없이 Live Activity 세션을 활성화함.
  * 만보기 데이터 대신 SwiftUI의 `Text(timerInterval: ...)`을 활용하여 **'⏳ 말씀과 동행한 시간 02:15:30'** 처럼 초 단위로 실시간 카운트업되는 타이머를 띄움.
  * 이를 통해 권한이 없는 상태에서도 화면의 Live 상태를 지속 유지하여 심사 가이드라인을 완벽하게 충족함.

## 10. 수익화 및 번역본 저작권 대응 전략
사업자 등록증이 없는 1인 개인 개발자 환경에서, 타사의 저작권(개역개정 등) 침해 리스크 없이 합법적으로 라이선스 비용을 마련하는 모금 전략.

* **수익 창출 수단:** 애플 공식 앱 내 결제(In-App Purchase) 중 **'소모성(Consumable) 아이템'** 활용. 개인 개발자 계정으로 애플을 통해 정산.
* **기획 방향:** 특정 기능을 유료로 판매하는 것이 아니라, **'앱 개발자 자율 후원(Tip)'** 형태로 구현하여 애플 가이드라인 준수.
* **안전한 모금 문구 가이드 (저작권 회피):**
  * 특정 번역본 명칭(예: 개역개정)을 상업적 모금에 직접 거론할 경우 상표권 및 저작권 분쟁의 소지가 있음.
  * *안전한 예시:* "현재 말씀 위젯은 무료 라이선스인 개역한글만 제공 중입니다. 1인 개발자를 후원해 주시면, 목표 금액 달성 시 **대중적으로 가장 많이 읽히는 현대어 번역본들**을 공식 라이선스로 체결하여 모든 분들께 무료로 업데이트해 드리겠습니다."
