따라서 **"정적 배치가 아닌 실시간 변화하는 이벤트"로 규정 가능**.

## ⚠️ 권장 개선 사항 (심사 실패 가능성: 낮음)

### A. HealthKit 권한 상태 정확한 판정

**영향도:** 중간 (권한 거절 시 사용자 경험)
**심사 위험도:** 낮음

```swift
// ios/Runner/AppDelegate.swift (Line 83-93) 개선안
if alreadyDetermined {
    let stepType = HKQuantityType.quantityType(forIdentifier: .stepCount)!
    let authStatus = healthStore.authorizationStatus(for: stepType)
    let isAuthorized = authStatus == .sharingAuthorized

    HealthKitService.shared.requestAuthorization { success in
        startWithAuth(isAuthorized)  // 실제 권한 상태 반영
    }
} else {
    startWithAuth(false)
}
```

**이유:** 권한 거절 시에도 Live Activity가 "타이머 모드"로 정상 작동해야 함.

This app features an 8-hour daily Live Activity session for
Scripture users. The key technical aspects:

   - Primary: Real-time step count via HealthKit API

   - Start a session and accept HealthKit permission
   - Verify steps update in real-time on lock screen
   - Deny HealthKit permission on a fresh device and verify

   - Session auto-terminates after 8 hours (iOS system behavior)

5. **Custom Photo Background:**
   - Users can select a custom image as widget background
   - Images are resized to 360px max and compressed at 0.5 quality
   - This prevents exceeding Live Activity memory limits (15-30MB)
   - Works in both light and dark themes

The app strictly adheres to Apple's Live Activities Guidelines and
leverages HealthKit only for legitimate user value (tracking activity
during morning routines).
